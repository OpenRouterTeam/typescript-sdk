# Example Tools

Production-ready tool implementations that demonstrate real-world patterns. Each example is copy-paste ready with complete type definitions.

## Available Examples

### [Weather Tool](/docs/guides/examples/weather-tool)

Classic introductory tool with external API integration.

- Input/output validation with Zod
- Error handling patterns
- Environment variable setup

```typescript
const weatherTool = tool({
  name: 'get_weather',
  description: 'Get current weather for any city',
  inputSchema: z.object({ city: z.string() }),
  outputSchema: z.object({ temperature: z.number(), conditions: z.string() }),
  execute: async ({ city }) => fetchWeather(city),
});
```

---

### [Database Search](/docs/guides/examples/database-search)

Generator tool with real-time progress streaming.

- Progress events during execution
- Pagination handling
- Search result aggregation

```typescript
const searchTool = tool({
  name: 'search',
  inputSchema: z.object({ query: z.string() }),
  eventSchema: z.object({ progress: z.number() }),
  outputSchema: z.object({ results: z.array(z.string()) }),
  execute: async function* ({ query }) {
    yield { progress: 50 };
    return { results: await search(query) };
  },
});
```

## Usage Pattern

Examples follow this common structure:

```typescript
import { OpenRouter, tool } from '@openrouter/sdk';
import { z } from 'zod';

const openrouter = new OpenRouter({
  apiKey: process.env.OPENROUTER_API_KEY,
});

// Create tool
const myTool = tool({
  name: 'tool_name',
  description: 'What the tool does',
  inputSchema: z.object({ /* inputs */ }),
  outputSchema: z.object({ /* outputs */ }),
  execute: async (params) => {
    // Implementation
  },
});

// Use with callModel
const result = openrouter.callModel({
  model: 'openai/gpt-4o-mini',
  input: 'Use the tool to...',
  tools: [myTool],
});

const text = await result.getText();
```
